# 劇迷戰力圖 + 成就徽章 規則

> **本檔為「你是什麼樣的劇迷?」戰力圖與成就徽章的權威規則書。** 規則由使用者訂死、**隨時可能要求更改**;每次改規則,務必同步更新（a）本檔（b）`data.js` 的 `personality()`／`badges()`（c）`js/mm-strings.js` 的文案。me.html（登入者）與 u.html／`js/u-view.js`（公開唯讀）共用同一套計算與渲染。

實作位置:計算 `data.js`、文案 `js/mm-strings.js`、戰力圖渲染 `js/mm-persona-radar.js`、徽章圖示 `js/mm-badge-icons.js`。

## 一、六邊形戰力圖

六軸,每軸 0–100 分。**共同規則:全部用對數前載**,絕不用線性——

```
fl(x, cap) = x <= 0 ? 0 : min(100, 100 * ln(1+x) / ln(1+cap))
```

亦即 `x = cap` 時剛好 100 分;頭幾筆成長快、越後越難,讓剛開始記錄的使用者就有感。含子成分（如跨國足跡的洲／國）也各自套 `fl` 再加權。

| # | 軸名 | i18n key | 計算 | 封頂（=100 分）| 事件文案 ev |
|---|---|---|---|---|---|
| 1 | 劇院常客 | `pr_volume` | `fl(total, 40)` | 總場次 40 場 | 「{n} 次」 |
| 2 | 嘗鮮作品 | `pr_explorer` | `fl(unique, 30)` | 不同作品 30 部 | 「{n} 部不同作品」 |
| 3 | 劇種涉獵 | `pr_genre` | `fl(dTags, 5)` | 不同劇種傳統 5 種 | 「{n} 種」 |
| 4 | 跨國足跡 | `pr_global` | `fl(洲數,5)*0.4 + fl(國數,10)*0.6` | 5 洲 + 10 國 | 「{n} 國 · {m} 洲」 |
| 5 | 狂粉劇迷 | `pr_loyal` | `fl(maxRep-1, 6)` | 同一齣刷 7 次 | 「最多 {n} 刷」 |
| 6 | 音樂劇齡 | `pr_age` | `fl(span, 30)` | 年跨度 30 年 | 「橫跨 {n} 年」 |

- **跨國足跡**權重:洲占 40%、國占 60%（國比洲難達成故權重高;非洲幾乎無音樂劇→5 洲即視為滿）。
- **狂粉劇迷**用 `maxRep-1`:沒重看過（maxRep=1）→ 0 分;同一齣刷 7 次（maxRep=7 → `fl(6,6)`）→ 100。
- 軸名歷史別名（i18n key 不變,僅顯示字改過）:劇院常客←觀劇量;嘗鮮作品←探索者;劇種涉獵←世界劇種通;跨國足跡←環球玩家／環球足跡;狂粉劇迷←忠誠鐵粉;音樂劇齡←劇齡／資深老手。

### 渲染行為（`mm-persona-radar.js`）

- **0 場**:不畫圖,只給鼓勵句（`p_none`）。
- **≥1 場**:畫六邊形。頂點半徑設 floor（約 12%）防止低分塌成一點,但**軸標籤、分數、清單仍顯示真實值**（不因 floor 而失真）。
- **<5 場**:圖下方加成長提示（`p_grow`）。
- **配色**:六軸各異色（紅／青綠／草綠／深藍／紫／洋紅,刻意避開品牌金,彼此不相近）;頂點圓點、軸標籤、下方清單編號同色對應。
- **級距線**:疊 25／50／75 格線,讓人看得出分數落在哪一段。
- 依主題上色（亮=香檳金箔／暗=靛藍月光）。

### 驗算基準（danny 真實資料）

27 場／17 部／2 劇種／4 國／4 洲／最高 5 刷／橫跨 21 年 →
劇院常客 90／嘗鮮作品 84／劇種涉獵 61／跨國足跡 76／狂粉劇迷 83／音樂劇齡 90。
**改任何公式後,務必用這組數字回歸驗算。**

## 二、成就徽章（17 族）

`data.js` 的 `badges()`。每族有多級**續階門檻**（`tiers`）,讓高活躍者永遠有下一級可追。回傳 `{key, value, tier, reached, next, extra}`,`tier=-1` 為未解鎖。

| key | 意義 | 門檻 tiers |
|---|---|---|
| `first` | 首場（附首演年）| [1] |
| `shows` | 總場次 | [5,10,25,50,100,200] |
| `countries` | 國家數 | [2,3,5,8] |
| `cities` | 城市數 | [3,8,15,25] |
| `works` | 不同作品數 | [10,25,50,100] |
| `devotee` | 同一齣最高刷數（附劇名）| [3,5,10] |
| `double` | 一日看兩場（雙刷日）數 | [1,3,5] |
| `streak` | 最長連續年數 | [2,3,5] |
| `fivestar` | 五星評分數 | [3,10,25] |
| `reviews` | 寫心得筆數 | [3,10,25] |
| `faves` | 標最愛數 | [3,10,20] |
| `traditions` | 不同劇種傳統數 | [2,3,4] |
| `regular` | 同一劇院最多場 | [3,5,10] |
| `marathon` | 單月最多場 | [3,5,8] |
| `veteran` | 年跨度 | [5,10,15] |
| `crosscity` | 同一齣跨幾城看 | [2,3] |
| `watchlist` | 未來（口袋）場次數 | [1,3,5] |

### 徽章視覺

- **金屬飾面三級**（`metal(tier)`,me.html／u-view.js）:`tier 0 → 銅`、`tier 1 → 銀`、`tier 2 以上 → 金`。即解鎖門檻雖 1~6 級不等,但顯示的金屬僅銅／銀／金三種,越高一律金。
- 飾面為 conic-gradient 金屬環（`css/me-v2.css`）;**金級**額外有光暈 box-shadow + `✦` 星芒閃爍動畫（`.badge.gold::before/::after`）。
- 圖示為手繪墨線 SVG（`js/mm-badge-icons.js`,布幕舞台／票券／地球／望遠鏡等,透明底,徽章 pill 內約 30px 顯示;me 與 u 共用）。
- 只顯示已解鎖（`tier>=0`）徽章;文案由頁面 i18n（`bd_*` / `bd_n_*` key）渲染,數字精確不模糊。
